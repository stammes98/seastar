The Homing Process

To begin homing, navigate to /home/seastar/seastar_ik/amMotor with the terminal. Once there, type "make run". This will compile and execute the terminal program used to interface with the applied motion motors. If it throws an error upon running, edit line 46 of main.cpp in the folder. Typing "make run" will recompile it and run it in the same step.

Once the terminal is running, we can interface with the motors using their Q commands. The first commands you'll want to send are 0MD 1MD and 2MD - these disable motors 0, 1, and 2. This will allow you to rotate the motors to the homed position. Once there, run the commands 0SP0, 1SP0, and 2SP0 individually. This manually tells the motor "this position is zero" - once done, that's it! No need to re-enable the motors, the motor_state_manager node will do that itself.
